关系代数

  1. 关系代数的运算对象是关系，运算结果不是关系。 {.quiz .false}

    关系代数的运算结果还是关系。

  2. 关系代数分为传统的集合运算和专门的关系运算，则专门的关系运算有哪些？ {.quiz .multi}

    • 选择。 {.correct}
    • 笛卡尔积。
    • 连接。 {.correct}
    • 除。 {.correct}
    • 投影。 {.correct}
      {.options}

      传统的集合运算有并，差，交，笛卡尔积。

  3. 关系的差运算，对于两个关系R和S，R-S和S-R一定是等价的。 {.quiz .false}

    关系的差运算有顺序，R-S与S-R一般不等价。

  4. 关系代数中，表示不等于的运算符是!=。 {.quiz .false}

    表示不等于的运算符只能是\<>。

  5. 用关系代数查询信息安全专业的全体学生。[]{.gap} {.quiz .fill}

    常见的在选择运算的条件表达式的运算符有>(大于),>=(大于等于),<(小于),<=(小于等于),=(等于),\<>(不等于),$\lnot$(非),$\land$(与),$\lor$(或)。

SQL语言

:::warning

本博文涉及的表有学生表Student、课程表Course、学生选课表SC。
Student($\underline{Sno}$,Sname,Ssex,Sbirthdate,Smajor)
学生($\underline{学号}$,姓名,性别,出生日期,专业)

Course($\underline{Cno}$,Cname,Ccredit,Teacher)
课程($\underline{课程号},课程名,学分,授课老师$)

SC($\underline{Sno}$,Cno,Grade,Semester)
学生选课($\underline{学号}$,课程号,成绩,选修学期)

:::

  1. 查询学号为20180003的学生的详细情况。 []{.gap} {.quiz .fill}

    SELECT * FROM Student 
WHERE Sno='20180003';

  2. 查询所有姓刘的学生的姓名、学号和性别。 []{.gap} {.quiz .fill}

    SELECT Sname, Sno, Ssex FROM Student 
WHERE Sname LIKE '刘%';

  3. 如果课程号长度为5个字符，查询课程号为81开头，最后一位是6的课程名称和课程号。 []{.gap} {.quiz .fill}

    SELECT Cname, Cno FROM Course 
WHERE Cno LIKE '81__6';

  4. 查询所有不姓刘的学生的姓名、学号和性别。 []{.gap} {.quiz .fill}

    SELECT Sname, Sno, Ssex FROM Student
WHERE Sname NOT LIKE '刘%';

  5. 查询DB_Design课程的课程号和学分。 []{.gap} {.quiz .fill}

    SELECT Cno, Ccredit FROM Course
WHERE Cname = 'DB\_Design' ESCAPE'\';

  6. 将学生20180001的出生日期改成2001年3月18日。 []{.gap} {.quiz .fill}

    UPDATE Student
SET Sbirthdate='2001-03-18'
WHERE Sno='20180001';

  7. 将2020年第1学期选修81002课程的所有学生的成绩减少5分。 []{.gap} {.quiz .fill}

    UPDATE Student
SET Grade=Grade-5
WHERE Semester='20201' AND Cno='81002';

  8. 将计算机科学与技术专业学生的成绩置零。 []{.gap} {.quiz .fill}

    UPDATE SC
SET Grade=0
WHERE Sno IN (
SELECT Sno From Student
WHERE Smajor='计算机科学与技术'
);

  9. 建立信息管理与信息系统专业学生的视图，并要求进行插入、修改和删除操作时，仍需保证该视图只有信息管理与信息系统专业的学生。 []{.gap} {.quiz .fill}

    CREATE VIEW xinxiguanli
AS
SELECT * FROM Student
WHERE Smajor='信息管理与信息系统'
WITH CHECK OPTION;

  10. 按照前文信息要求，创建SC表，说明Sno, Cno, Grade属性不允许取空值。 []{.gap} {.quiz .fill}

    CREATE TABLE SC
(
Sno CHAR(8) NOT NULL,
Cno CHAR(5) NOT NULL,
Grade SMALLINT NOT NULL,
Semester CHAR(5),
PRIMARY KEY(Sno, Cno)
)

  11. 建立学院表School，要求学院名称SHname列取值唯一，且要求为变长的字符，长度最大为40，学院编号SHno列为主码，要求为8个字符长度，另外还有属性SHfounddate表示建立日期。 []{.gap} {.quiz .fill}

    CREATE TABLE School
(
SHno CHAR(8) PRIMARY KEY,
SHname VARCHAR(40) UNIQUE,
SHfounddate DATE
)

  12. 按照前文信息要求，创建Student表，且要求性别属性为6字符长度且只允许取’男’或’女’，学号属性为8个字符长度且为主码，姓名属性为20个字符长度且不为空，专业属性为变长的字符串，最长为20。 []{.gap} {.quiz .fill}

    CREATE TABLE Student
(
Sno CHAR(8) PRIMARY KEY,
Sname CHAR(20) NOT NULL,
Ssex CHAR(6) CHECK(Ssex In('男','女')),
Sbirthdate DATE,
Smajor VARCHAR(20)
)

  13. 检索学生S1所选修全部课程的课程号与成绩，并按照成绩逆序排序。 []{.gap} {.quiz .fill}

    SELECT Cno, Grade FROM SC
WHERE Sno = (
SELECT Sno FROM Student
WHERE Sname='S1'
)
ORDER BY Grade DESC。

  14. 检索同时选修了至少两门课程的学生学号与姓名。 []{.gap} {.quiz .fill}

    SELECT Sno, Sname FROM Student
WHERE Sno IN (
SELECT Sno FROM SC 
 GROUP BY SNO
   HAVING COUNT(*)>=2
);

  15. 检索至少选修LIU老师所授课程中一门课程的女学生姓名、课程名及成绩。 []{.gap} {.quiz .fill}

    SELECT a.Sname, c.Cname, b.Grade FROM Student a, SC b, Course c
WHERE a.Sno=b.Sno AND b.Cno=c.Cno AND a.Ssex='女' AND b.Teacher LIKE 'LIU%';

  16. 检索ZHANG老师所授课程的每门课程的平均成绩。 []{.gap} {.quiz .fill}

    SELECT AVG(SC.Grade) FROM SC, Course
WHERE SC.Cno=Course.Cno AND Course.Teacher LIKE 'ZHANG%'
GROUP BY SC.Cno;

  17. 检索所有女学生的学号和姓名，并按学号逆序排序。 []{.gap} {.quiz .fill}

    SELECT Sno, Sname FROM Student
WHERE Ssex='女'
ORDER BY Sno DESC;

  18. 检索同时选修了至少三门课程的学生学号与姓名。 []{.gap} {.quiz .fill}

    SELECT Student.Sno, Student.Sname FROM Student, SC
WHERE Student.Sno=SC.Sno
GROUP BY Student.Sno
HAVING COUNT(SC.Cno)>=3;

数据库设计

  1. 数据库设计的基本步骤有 [需求分析阶段]{.gap}、[概念结构设计阶段]{.gap}、[逻辑结构设计阶段]{.gap}、[物理结构设计阶段]{.gap}、[数据库实施阶段]{.gap}、[数据库运行和维护阶段]{.gap}。 {.quiz .fill}

  2. 数据字典是进行详细的 [数据收集]{.gap} 和 [数据分析]{.gap} 之后所获得的主要成果。它是关于数据库中对数据的 [描述]{.gap}，即元数据，而不是数据本身。 {.quiz .fill}

  3. 数据字典在需求分析阶段被建立，在数据库设计过程中不断修改、充实和完善。 {.quiz .true}

  4. 数据字典通常包括哪些部分？ {.quiz .multi}

    • 数据项。 {.correct}
    • 数据结构。 {.correct}
    • 数据流。 {.correct}
    • 数据存储。 {.correct}
    • 处理过程。 {.correct}
      {.options}

  5. 数据项是数据的最小组成单位，若干个数据项能组成一个数据结构。 {.quiz .true}
